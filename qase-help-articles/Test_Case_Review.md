# Test Case Review

***⚠️Review is available in [Business](https://help.qase.io/en/articles/5563727-business-plan) and [Enterprise](https://help.qase.io/en/articles/6640055-enterprise-plan) subscriptions.***

In Qase, it is possible to ensure that the [Test Cases](https://help.qase.io/en/articles/5563704-test-cases) being created are first verified by someone superior, be it a QA team manager, supervisor, CTO, or else. In the [Project Settings](https://help.qase.io/en/articles/5563706-projects#h_e70fd5f298), you can define several options regarding Test Case Review:

[![](https://downloads.intercomcdn.com/i/o/609559905/86d52d438bc3802945719c4f/image.png?expires=1771427700&signature=688a630dae394c1d4bfff96e30cff3d570090b5986ee575efbdc63ca54a2bc86&req=ciAuE8x3lIFaFb4f3HP0gIUSfcXok0yKWa9qwRS75r69SyzBqFxfLcCXwJWZ%0AisOmQsyv8mFgsGvdFg%3D%3D%0A)](https://downloads.intercomcdn.com/i/o/609559905/86d52d438bc3802945719c4f/image.png?expires=1771427700&signature=688a630dae394c1d4bfff96e30cff3d570090b5986ee575efbdc63ca54a2bc86&req=ciAuE8x3lIFaFb4f3HP0gIUSfcXok0yKWa9qwRS75r69SyzBqFxfLcCXwJWZ%0AisOmQsyv8mFgsGvdFg%3D%3D%0A)

|  |  |
| --- | --- |
| Review enabled | If selected, you and your teammates will have the option to send a test case to review. |
| Review is mandatory | If selected, any changes submitted for a test case cannot be updated unless it’s sent to review. |
| Self Merge | This option permits the initiator of the review to merge the changes requested in the review. |
| Approvals required | In order to merge the changes, the number of approvals (determined by the Administrator/Owner) will have to be met. |

## For Reviewers:

If you are a reviewer and not the author who sent the Test Case to Review. You will be able to see five options on the right-hand side of the page, which are - Merge, Approve, Request Changes, Edit, and Decline.

[![](https://downloads.intercomcdn.com/i/o/wsaz8vex/1304709589/700522da4eb11a94557fc6fef5c7/75299.png?expires=1771427700&signature=46f0867b68c3b88e2ecdf2f5a7acaf9f045d3502ac0b376809fcaeb14a33778c&req=dSMnEs5%2BlIRXUPMW1HO4zdsty8wPliUPKHxKwcTstaj8qhp7MpScDDItxF4R%0AKjH62R2%2B%2FvDSi5tGqyk%3D%0A)](https://downloads.intercomcdn.com/i/o/wsaz8vex/1304709589/700522da4eb11a94557fc6fef5c7/75299.png?expires=1771427700&signature=46f0867b68c3b88e2ecdf2f5a7acaf9f045d3502ac0b376809fcaeb14a33778c&req=dSMnEs5%2BlIRXUPMW1HO4zdsty8wPliUPKHxKwcTstaj8qhp7MpScDDItxF4R%0AKjH62R2%2B%2FvDSi5tGqyk%3D%0A)

When you click on *Merge*, the suggested changes are accepted and updated to the test case.

*Approve* allows you to cast your vote for the change while other reviewers still have the chance to look at the suggested updates.

*Request Changes* allows you to update your status for the review as someone who has requested changes. This feature is a visual indicator to the other reviewers that you’ve requested for changes.

*Edit* allows you to suggest further changes to the review.

*Decline* allows you to decline the review request.

## For Authors:

If you’re the person who created the review, you will be able to Merge the review yourself if the following two conditions are met:

1. If the option to self-merge is enabled in the settings
2. If the required approvals number is not set

Here’s what the page would look like for the author if the conditions stated above are met. You will have three options available to you: Merge, Edit, and Decline.
